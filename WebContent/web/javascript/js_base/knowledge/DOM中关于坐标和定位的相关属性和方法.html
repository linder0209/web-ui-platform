window对象中坐标和位置的相关属性

包含以下几个属性

浏览器距离屏幕y坐标值：window.screenTop ，firefox中用window.screen.top代替

浏览器距离屏幕x坐标值：window.screenLeft ，firefox中用window.screen.left代替 

屏幕的高度：window.screen.height  

屏幕的宽度：window.screen.width  

屏幕可见工作区y坐标：window.screen.availTop

屏幕可见工作区x坐标：window.screen.availLeft

屏幕可用工作区高度：window.screen.availHeight  

屏幕可用工作区宽度：window.screen.availWidth

看下面的测试代码

console.log(window.screenTop);//距离屏幕y坐标值，firefox不支持该属性
console.log(window.screenLeft); //距离屏幕x坐标值，firefox不支持该属性
console.log(window.screen.top);//firefox中用该属性代替window.screenTop
console.log(window.screen.left);//firefox中用该属性代替window.screenLeft
console.log(window.screen.height); //屏幕的高度
console.log(window.screen.width); //屏幕的宽度
console.log(window.screen.availTop); //屏幕可见工作区y坐标
console.log(window.screen.availLeft); //屏幕可见工作区x坐标
console.log(window.screen.availHeight); //屏幕可见工作区高度，即除去任务栏的高度
console.log(window.screen.availWidth);//屏幕可见工作区宽度，即除去任务栏的宽度

通过以上运行结果，下面分组总结

document.body.clientWidth 、 document.body.clientHeight 、 document.body.clientLeft、 document.body.clientTop

在标准兼容模式下，即document.compatMode值为CSS1Compat，各大浏览器返回的值是一样的。 document.body.clientWidth返回的值为body的width+内边距，的。 document.body.clientHeight返回的值为body的height+内边距、 document.body.clientLeft为border-left-width的值、 document.body.clientTop为border-top-width的值。看w3c中的解释。

但在document.compatMode值为BackCompat下，是返回页面可视化高宽度，不包括滚动条，但在ie下还不包括外边距

document.body.offsetWidth 、 document.body.offsetHeight 、 document.body.offsetLeft、 document.body.offsetTop

这几个属性与offsetParent 有关，而body除外。详细看w3c的解释

document.body.scrollWidth 、 document.body.scrollHeight 、 document.body.scrollLeft、 document.body.scrollTop

这几个属性与滚动元素有关。详细看w3c的解释

其他DOM元素对象中坐标和位置的相关属性

在第二节中介绍body的相关属性实际上就是DOM元素中的一个特例，下面来看所有DOM元素的这些属性。

我们结合官方提供的文档来分析一下这些属性。这些属性可以分为三组：client、offset和scroll，首先看client

clientWidth 、 clientHeight 、 clientLeft、 clientTop

The clientTop, clientLeft, clientWidth, and clientHeight attributes must return zero if the element does not have any associated CSS layout box or if the CSS layout box is inline.

翻译为：如果元素没有关联的css layout box（即本身没有内容，也没有边框和边距，祖先元素也没有高宽度）或者该元素为级联元素（行内元素），则这些属性返回0。

clientTop 返回元素的'border-top-width'值

clientLeft 返回元素的'border-left-width'值

clientWidth 返回元素的可视宽度（包括内边距padding，但不包括边框、外边距和滚动条），即元素的宽度+padding-滚动条宽度

clientHeight 返回元素的可视高度（包括内边距padding，但不包括边框、外边距和滚动条），即元素的高度+padding-滚动条宽度

以上属性firefox、chrome在标准模式和怪异模式下显示都是一样的。ie在怪异模式下计算结果为，clientWidth：元素的宽度减去边框再减去滚动条；clientHeight：元素的高度减去边框再减去滚动条。ie在正常模式下与firefox一样。clientLeft和clientTop在标准和怪异模式下都与firefox一样。

offsetParent、offsetWidth 、 offsetHeight 、 offsetLeft、 offsetTop

offsetParent在以下情况下返回null

  • The element does not have an associated CSS layout box.
  • The element is the root element.
  • The element is the HTML body element.
  • The element's computed value of the 'position' property is fixed.

对于最后一种情况，firefox下也可以返回offsetParent

offsetParent返回最近祖先元素的规则需要判断祖先元素，规则为：

  • The computed value of the 'position' property is not static.
  • It is the HTML body element.
  • The computed value of the 'position' property of the element is static and the ancestor is one of the following HTML elements: td, th, or table.

offsetLeft 返回offsetParent的padding边缘到该元素的border边缘的y-coordinate距离

offsetTop 返回offsetParent的padding边缘到该元素的border边缘的x-coordinate距离

如果offsetParent为空，则返回“the coordinate of the left border edge of the first CSS layout box associated with the element”

offsetWidth和offsetWidth返回元素的大小加上内边距和边框，即有以下简单计算

offsetWidth = clientWidth + borderLeftWidth +borderRightWidth + srollWidth = width + border +padding

offsetHeight = clientHeight + borderTopWidth +borderBottomWidth + srollWidth = height + border +padding

注意ie的怪异模式跟标准模式不一样

scrollIntoView方法，scrollWidth 、 scrollHeight 、 scrollLeft、 scrollTop

scrollLeft和scrollTop简单的理解就是元素滚动条的相对元素本身的移动坐标，详细解释参考w3c，该属性是可写的

scrollWidth：Return the computed value of the 'padding-left' property, plus the computed value of the 'padding-right', plus the content width of the element.

scrollHeight：Return the computed value of the 'padding-top' property, plus the computed value of the 'padding-bottom', plus the content height of the element.

注意：1、需要减去scroll的宽度，2、是指内容的实际宽度，而非可见宽度

注意ie的怪异模式跟标准模式不一样

scrollIntoView方法的作用是让该元素的最左边、最上边靠近父元素的border边缘
